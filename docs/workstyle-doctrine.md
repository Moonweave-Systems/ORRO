# ORRO Workstyle Doctrine

ORRO should encode practical developer judgment before automation.

Rules:

- Prefer the smallest safe workflow.
- Do not run agents just because agents are available.
- Do not use proofrun for review-only tasks.
- Do not use proofrun for verification-only tasks.
- Use proofcheck for evidence verification.
- Stop instead of guessing.
- Risky changes require human review.
- Use bounded auto only after proofrun evidence exists.
- Skill text, transcripts, model confidence, and prose do not raise assurance.

```text
Depone verifies; witnessd executes; ORRO exposes the workflow.
```
